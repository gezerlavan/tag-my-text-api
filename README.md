# Tag My Text API

A simple yet powerful Express.js API that generates relevant tags for any given text using Google's Gemini AI model.

## Overview

This API provides a single endpoint that accepts text input and returns 3â€“6 relevant tags generated by Google's Gemini 2.5 Flash model. It's designed to be fast, cost-effective, and easy to integrate.

## Features

- **AI-Powered Tagging**: Uses Google Gemini 2.5 Flash for intelligent tag generation
- **Simple REST API**: Single POST endpoint for easy integration
- **Error Handling**: Graceful fallbacks and meaningful error messages
- **Environment Configuration**: Secure API key management via environment variables

## Prerequisites

- Node.js (v16 or higher)
- A Google API key with Gemini access
- npm or yarn package manager

## Installation

1. Clone or download this repository
2. Install dependencies:

```bash
npm install
```

3. Create a `.env` file in the root directory:

```env
GEMINI_API_KEY=your_google_api_key_here
PORT=3000
```

## Usage

Start the server:

```bash
node server.js
```

The API will be available at `http://localhost:3000`

### API Endpoint

**POST** `/tag`

#### Request Body

```json
{
  "text": "Your text content here"
}
```

#### Response

```json
{
  "tags": ["tag1", "tag2", "tag3"]
}
```

#### Example

```bash
curl -X POST http://localhost:3000/tag \
  -H "Content-Type: application/json" \
  -d '{"text": "I had a wonderful day at the beach with my friends"}'
```

Response:
```json
{
  "tags": ["beach", "friends", "fun", "vacation"]
}
```

## Error Handling

- **400 Bad Request**: Missing `text` field in request body
- **500 Internal Server Error**: AI tag generation failed

## Project Structure

- `server.js` - Express server setup and API endpoint definition
- `ai.js` - AI integration with Google Gemini and tag generation logic
- `package.json` - Project dependencies and metadata

## Dependencies

- **express**: Web server framework
- **@google/genai**: Google Generative AI client library
- **dotenv**: Environment variable management

## Notes

- The API uses Gemini 2.5 Flash model for optimal speed and cost efficiency
- Tags are automatically normalized to lowercase
- Empty or invalid responses are handled gracefully with fallback to empty array
- The model is instructed to return JSON array format for reliable parsing
